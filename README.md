# 🚖 Taxi Service Backend

Микросервисная платформа для заказа такси. Позволяет пассажирам создавать поездки, водителям — принимать их и менять статусы, а также оценивать завершённые поездки. Фоновая обработка уведомлений реализована через пул воркеров с использованием базы данных как очереди.

## 📦 Основные возможности

- Регистрация и аутентификация пассажиров и водителей (JWT).
- Создание поездки с автоматическим назначением свободного водителя.
- Изменение статуса поездки: `CREATED → ACCEPTED → STARTED → COMPLETED`.
- Оценка поездки (1–5 звёзд) после завершения.
- История поездок пассажира.
- Статистика по поездкам за день (количество, средняя цена).
- Асинхронные уведомления (создание, смена статуса, оценка) через очередь в PostgreSQL.

## 🧱 Архитектура

Система состоит из трёх микросервисов и одной общей базы данных:

| Сервис          | Порт | Описание |
|----------------|------|-----------|
| **user-service** | 8001 | Управление пассажирами и водителями, аутентификация, назначение свободного водителя (блокировки на уровне БД). |
| **trip-service**| 8002 | Создание поездок, управление статусами, рейтинг, статистика. Взаимодействует с user-service через RestTemplate. |
| **worker-service** | 8003 | Приём запросов на уведомления, обработка в фоне через пул воркеров (SKIP LOCKED). |

**База данных:** PostgreSQL 18.3, три отдельные базы (`user_service`, `trip_service`, `worker_service`).  
**Миграции:** Liquibase (changelog в `migrations/`).  
**Авторизация:** JWT, фильтры проверяют токен во всех запросах к `/api/*` (кроме публичных эндпоинтов регистрации и логина).

## 🚀 Запуск

1. Клонируйте репозиторий и перейдите в корневую директорию проекта.
2. Создайте файл `.env` в корне (см. пример ниже).
3. Выполните сборку и запуск:

```bash
docker compose up -d
```

При первом запуске будут выполнены:
- Создание баз данных (скрипт init.sh)
- Миграции Liquibase для всех трёх сервисов (migrate.sh)
- Сборка и запуск каждого микросервиса

Сервисы будут доступны по адресу:
- User Service: http://localhost:8001
- Trip Service: http://localhost:8002
- Worker Service: http://localhost:8003

Пример .env файла:
```.env
POSTGRES_USER=postgres
POSTGRES_PASSWORD=postgres123
POSTGRES_DB=postgres

DB_USERNAME=taxi_user
DB_PASSWORD=taxi_pass

JWT_SECRET=your_super_secret_key_for_jwt_256bit_minimum

PASSENGER_PASSWORD=pass123
DRIVER_PASSWORD=drive123
```

## 🧪 Тестирование
End-to-End сценарий
Скрипт e2e_test_full_program_success_script.sh выполняет полный цикл:
- регистрация пассажира и водителя
- логин, получение токенов
- создание поездки, принятие, начало, завершение
- выставление оценки
проверка уведомлений, списков пользователей и статистики

Запуск (убедитесь, что контейнеры запущены, после запуска может потребоваться несколько секунд подождать перед отправкой запросов):
```bash
chmod +x e2e_test_full_program_success_script.sh
./e2e_test_full_program_success_script.sh
```

Скрипт run_tests.sh содержит более 20 проверок:
- аутентификация (401 при отсутствии/неверном токене)
- дублирование email/телефона при регистрации
- создание поездки (успех, несуществующий пассажир, отсутствие свободных водителей)
- смена статусов, рейтинг (валидный/невалидный, незавершённая поездка)
- статистика, уведомления, назначение водителя

Запуск:
```bash
chmod +x run_tests.sh
./run_tests.sh
```
